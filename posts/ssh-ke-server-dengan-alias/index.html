SSH ke server dengan alias

By Sehan on July 12, 2021
tips ssh

Mungkin bagi kamu yang kesehariannya bolak-balik ke server biasanya lupa mengingat IP server, user di server tersebut, dan belum lagi berbagai port yang unik-unik.

Kali ini saya akan share cara native dari konfigurasi OpenSSH, tapi sebelum itu diasumsikan kamu udah setup SSH di komputer kamu ya. Kalo belum, kamu bisa baca disini untuk cara setup SSH di komputer kamu.

Lokasi file konfigurasi SSH

File konfigurasi OpenSSH bernama config, dan disimpan di direktori ~/.ssh. By default, file konfigurasi SSH mungkin ga ada, jadi kamu mungkin perlu membuatnya menggunakan perintah berikut terlebih dulu.

$ touch ~/.ssh/config

Format file konfigurasi SSH

File konfigurasi SSH mempunyai struktur seperti berikut:

Host host1
    SSH_OPTION value
    SSH_OPTION value

Host host2
    SSH_OPTION value

Isi file konfigurasi SSH diatur ke dalam bagian. Setiap bagian dimulai dengan direktif Host dan berisi opsi SSH khusus yang digunakan saat membuat koneksi dengan server SSH jarak jauh. Indentasi ga diperluin tapi disarankan karena membuat file lebih gampang dibaca. Untuk daftar lengkap opsi ssh yang tersedia lihat disini.

FYI: File konfigurasi SSH juga dibaca oleh program scp, sftp, dan rsync.

Contoh konfigurasi SSH

Biasanya, saat menghubungkan ke server via SSH, kamu harus tau nama user, nama host, dan port. Misalnya, untuk masuk sebagai user ubuntu ke host 107.40.13.145 pada port 8001 dari console, kamu harus ngetik:

$ ssh ubuntu@107.40.13.145 -p 8001

Untuk terhubung ke server menggunakan opsi yang sama seperti yang diketikan pada command di atas, cukup dengan mengetik ssh mysql-prod misalnya, masukkan konfigurasi berikut ke file ~/.ssh/config kamu:

Host mysql-prod
    HostName 107.40.13.145
    User ubuntu
    Port 8001

Sekarang ketika kamu ngetik ssh mysql-prod dengan gaya di console, ssh akan membaca file konfigurasi dan menggunakan detail koneksi yang ditentukan untuk host mysql-prod. Gampang kan? kalo ada server baru lagi, kamu tinggal nambah bagian baru di file konfigurasi ssh nya. Kalo masih ada yang bingung, tanya aja di komentar ya.
